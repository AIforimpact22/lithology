How the workflow comes together

The process below mirrors the data pipeline powering this viewer.

01 Collect geological PDFs

Gather source lithology documents and store them in the repository alongside the Excel workbook reference.

02 Parse workbook intervals

The Flask backend reads Geological Profiles.xlsx, extracting interval depths and descriptions for each tab.

03 Merge PDFs & tables

Each entry is matched to its lithology table so you can move from the PDF to structured data instantly.

04 Deliver insights

The front-end renders rich cards with download links and inline tables for quick comparison across boreholes.
